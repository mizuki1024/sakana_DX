# MVP Requirements for First PoC

この文書は、1社目PoCに必要なMVP要件を定義する。

## 対象顧客

小規模な漁市場、漁協、水産卸、仲卸業者。

想定課題:

- 紙やExcelで水揚げ・販売・請求を管理している。
- 仕切書や請求書の作成に手作業が多い。
- ロット、買受人、仕入先、金額の確認に時間がかかる。
- 過去データの検索や集計が難しい。

## MVPの価値

水揚げ登録から販売、仕切書、請求書までを一気通貫で管理し、紙・Excel作業を減らす。

## 対象業務フロー

1. 仕入先から水揚げ情報を登録する。
2. ロット番号を発行する。
3. 買受人へ販売登録する。
4. 販売ごとに仕切書を確認・印刷する。
5. 月次または締め日ごとに請求書を作成する。
6. 請求書を確認・印刷する。
7. 必要に応じてCSVで出力する。

## 必須機能

### 認証

- 職員がログインできる。
- 未ログインユーザーは業務画面へアクセスできない。

### 水揚げ管理

- 水揚げを登録できる。
- 仕入先、魚種、重量、等級、保管場所を入力できる。
- ロット番号を自動発行できる。
- 水揚げ一覧を確認できる。
- 未販売・販売済み・請求済みの状態を確認できる。

### 販売管理

- 未販売の水揚げだけを販売登録に使える。
- 買受人と販売金額を登録できる。
- 二重販売を防止できる。
- 販売一覧と詳細を確認できる。

### 仕切書

- 販売詳細から仕切書を表示できる。
- 仕入先、ロット、魚種、重量、販売価格、手数料、仕切金額を表示できる。
- 印刷できる。

### 請求管理

- 買受人ごと、対象月または締め期間ごとに請求書を作成できる。
- 請求対象の売上を集計できる。
- 請求書一覧を確認できる。
- 請求書詳細を表示・印刷できる。
- 請求済み売上を再請求しない。

### マスタ管理

- 仕入先を登録・編集できる。
- 買受人を登録・編集できる。
- コード、名称、連絡先を管理できる。

### 出力

- 水揚げ一覧をCSV出力できる。
- 売上一覧をCSV出力できる。
- 請求一覧をCSV出力できる。

## 非機能要件

### データ保護

- 取引データは安易に物理削除しない。
- 重要操作は監査ログに残す。
- DBバックアップ方針を用意する。

### 操作性

- 登録フォームは現場で迷わず入力できる。
- 金額と日付は日本の業務表記に合わせる。
- 印刷帳票はA4で崩れない。

### 信頼性

- 金額計算は一貫したルールで行う。
- 二重登録や二重販売を防ぐ。
- エラー時にユーザーへ分かるメッセージを出す。

## MVP外

PoC初期では作らない。

- 決済連携
- 会計ソフト連携
- 高度な在庫管理
- 需要予測
- EC連携
- 完全なスマホアプリ
- 顧客ごとの自由帳票エディタ

## 未確定ルール

初回ヒアリングで確認する。

- 手数料率は固定か、仕入先・魚種・買受人ごとか。
- 消費税を請求書に含めるか。
- 税率、内税・外税、端数処理。
- 締め日は月末か、買受人別か。
- 仕切書番号と請求書番号の採番ルール。
- 請求書の宛名、住所、登録番号、振込先。
- 訂正・取消時の社内承認フロー。
- 既存Excelデータの移行要否。

## PoC完了条件

- 実データに近いサンプルで、水揚げから請求書印刷まで通せる。
- 顧客の現行帳票と見比べて、業務上の不足が整理されている。
- 1週間分または1締め分のデータを入力しても破綻しない。
- 顧客が月額利用または継続開発の判断をできる。
