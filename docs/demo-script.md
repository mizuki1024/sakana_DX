# Demo Script

この台本は、1社目候補に対して「漁市場デジタル台帳」を説明するための初回デモ用資料である。

## デモの目的

完成品を売り切ることではなく、相手の現場業務に合うかを確認し、有償PoCにつなげる。

## 想定時間

15分から30分。

## 冒頭

話す内容:

> 水揚げ、販売、仕切書、請求書の管理を紙やExcelから置き換える小規模市場向けの台帳システムです。今日は完成品の押し売りではなく、実際の業務に合うかを確認するために、一般的な流れを見ていただきたいです。

確認すること:

- 現在は紙、Excel、既存ソフトのどれで管理しているか。
- 水揚げから請求まで誰が担当しているか。
- 一番時間がかかっている作業は何か。

## デモの流れ

### 1. ログイン

見せる内容:

- 職員がログインする。
- 未ログインでは業務画面に入れない。

伝える価値:

- 担当者だけが取引データを扱える。

### 2. 水揚げ登録

見せる内容:

- 仕入先を選ぶ。
- 魚種、重量、等級、保管場所を入力する。
- ロット番号が発行される。

伝える価値:

- 紙台帳の記録を検索可能なデータに変える。
- ロット単位で後続の販売・請求とつながる。

質問:

- 今のロット番号や伝票番号の付け方はどうしているか。
- 等級やサイズの分類は何を使っているか。

### 3. 販売登録

見せる内容:

- 未販売の水揚げを選ぶ。
- 買受人を選ぶ。
- 販売金額を入力する。
- 販売済みに変わる。

伝える価値:

- 二重販売を防ぎやすくなる。
- 買受人別の売上集計につながる。

質問:

- 販売単価はkg単価か、ロット合計か。
- 分割販売は必要か。
- 入札・相対・競りの区分は必要か。

### 4. 仕切書

見せる内容:

- 販売詳細から仕切書を開く。
- 販売価格、手数料、仕切金額を確認する。
- 印刷する。

伝える価値:

- 手作業の転記を減らせる。
- 仕入先への説明資料をすぐ出せる。

質問:

- 手数料率は固定か。
- 仕切書の様式は現在決まっているか。
- 消費税や控除項目はあるか。

### 5. 請求書

見せる内容:

- 買受人ごとに対象期間の売上を集計する。
- 請求書を作成する。
- 請求済みとして管理する。

伝える価値:

- 月末や締め日の請求作業を短縮できる。
- 請求漏れや二重請求を防げる。

質問:

- 締め日は月末か、買受人別か。
- 請求書は郵送、手渡し、メールのどれか。
- 入金確認まで管理したいか。

## クロージング

話す内容:

> 今日見ていただいたものをベースに、御社の帳票や締め処理に合わせて1〜2か月の実証導入ができます。最初の導入先として、通常より範囲を絞った価格で、現場に合わせながら改善する形を想定しています。

提案:

- 無料ヒアリング
- 現行帳票の確認
- サンプルデータ投入
- 有償PoC提案

## ヒアリング質問

業務:

- 1日の水揚げ件数は何件程度か。
- 1日の販売件数は何件程度か。
- 仕入先と買受人は何社程度か。
- 締め処理は月何回か。

帳票:

- 現在の仕切書と請求書のサンプルを見せてもらえるか。
- 必須項目は何か。
- 社印、登録番号、振込先は必要か。

運用:

- 誰が入力し、誰が確認するか。
- 修正や取消は誰が承認するか。
- 過去データの移行は必要か。

価格:

- 初期導入費と月額のどちらが受け入れやすいか。
- 帳票カスタムに費用がかかることは問題ないか。

## PoC提案の目安

小さく始める場合:

- 期間: 1か月
- 初期費用: 5万〜15万円
- 範囲: 水揚げ、販売、仕切書、簡易請求

標準的に始める場合:

- 期間: 2か月
- 初期費用: 15万〜30万円
- 範囲: 帳票調整、請求締め、CSV、簡易マニュアル

PoC後:

- 月額: 1万〜5万円
- 帳票追加・データ移行・会計連携は別途見積もり

## 付録: デモ操作手順

### 事前準備

```bash
# サンプルデータを投入
npx prisma db seed

# アプリ起動
npm run dev
```

### テストアカウント

| ロール | メールアドレス | パスワード |
|--------|----------------|------------|
| 管理者 | admin@example.com | password123 |
| スタッフ | staff@example.com | password123 |

### 操作手順

#### 1. ログイン
- http://localhost:3000 にアクセス
- `admin@example.com` / `password123` でログイン

#### 2. 水揚げ登録
- サイドバー「水揚げ一覧」→「新規登録」
- 日付: 今日、仕入先: 山田水産、魚種: マグロ
- ロット番号: LOT-DEMO-001、重量: 150、等級: A
- 「登録」をクリック

#### 3. 売上登録
- サイドバー「売上一覧」→「新規登録」
- 水揚げ: LOT-DEMO-001、買受人: 東京水産市場
- 売上金額: 450000
- 「登録」をクリック

#### 4. 請求書作成
- サイドバー「請求書一覧」→「新規作成」
- 買受人: 東京水産市場、対象月: 今月
- プレビュー確認→「作成」
- 詳細画面で「印刷」をクリック

#### 5. ステータス変更
- 請求書詳細で「送付済みにする」をクリック
- 「入金済みにする」をクリック

#### 6. CSVエクスポート
- 各一覧画面で「CSVエクスポート」をクリック

#### 7. 権限の違い（任意）
- ログアウト→ `staff@example.com` でログイン
- 買受人一覧で削除ボタンが無効化されていることを確認
