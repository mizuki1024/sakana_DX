# Release Checklist

このチェックリストは、本番公開または顧客PoC前に確認する項目を定義する。

## Product

- [ ] 対象顧客とPoC範囲が明確である。
- [ ] 水揚げから請求までのデモが通る。
- [ ] 作る機能と作らない機能が説明できる。
- [ ] PoC完了条件が明確である。
- [ ] 価格、期間、サポート範囲が明確である。

## Domain

- [ ] 水揚げ、販売、仕切、請求の業務フローが文書化されている。
- [ ] 手数料率の扱いが明確である。
- [ ] 消費税、端数処理、内税・外税の扱いが明確である。
- [ ] 締め日と請求対象期間の扱いが明確である。
- [ ] 仕切書番号・請求書番号の採番ルールが明確である。
- [ ] 訂正・取消フローが明確である。

## Data

- [ ] 取引データを安易に物理削除しない設計になっている。
- [ ] 二重販売を防止している。
- [ ] 請求済み売上の再請求を防止している。
- [ ] 締め済みデータの編集制限がある。
- [ ] 金額計算の確定値を追跡できる。
- [ ] 監査ログの方針がある。

## Security

- [ ] 本番用の `NEXTAUTH_SECRET` を使っている。
- [ ] 本番用の `NEXTAUTH_URL` を設定している。
- [ ] DB接続情報をリポジトリに含めていない。
- [ ] 管理者と職員の権限差がある。
- [ ] 未ログインユーザーが業務APIへアクセスできない。
- [ ] 個人情報や取引情報を不要にログ出力していない。

## Operations

- [ ] 本番DBのバックアップ手順がある。
- [ ] 復旧手順を一度確認している。
- [ ] エラー確認方法がある。
- [ ] 問い合わせ窓口と対応時間を決めている。
- [ ] 障害時に顧客へ説明する手順がある。
- [ ] データ修正依頼の受付ルールがある。

## Frontend and UX

- [ ] 登録フォームの必須項目が分かりやすい。
- [ ] 入力エラーが日本語で表示される。
- [ ] 一覧に検索・絞り込みの導線がある。
- [ ] 金額、日付、重量の表記が現場向けである。
- [ ] 印刷時に不要なUIが出ない。
- [ ] A4印刷で帳票が崩れない。

## QA

- [ ] 水揚げ登録から請求書印刷までの正常系を確認した。
- [ ] 未ログイン時のアクセス制限を確認した。
- [ ] 二重販売できないことを確認した。
- [ ] 請求済み売上が再請求されないことを確認した。
- [ ] 締め後修正の扱いを確認した。
- [ ] 空データ、異常値、必須項目漏れを確認した。
- [ ] 帳票とCSVの内容を目視確認した。

## Sales and Onboarding

- [ ] デモ台本がある。
- [ ] サンプルデータがある。
- [ ] 初回ヒアリング質問がある。
- [ ] PoC提案の価格帯がある。
- [ ] 初期設定手順がある。
- [ ] 顧客への説明資料がある。

## Release Decision

PoCに出してよい条件:

- [ ] デモが止まらない。
- [ ] 請求・仕切の不足点を説明できる。
- [ ] データ消失リスクへの対策がある。
- [ ] 顧客に未完成部分を正直に説明できる。
- [ ] PoC中の改善対応範囲が決まっている。
